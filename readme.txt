CREDENCIALES MAS ABAJO.

Esta es una simulacion de una red social para la universidad de medicina.
Su principal funcion es poder compartir archivos relacionados con la universidad como resumenes
asi como fotos y videos.
Para poder simular una base de datos que guarde imagenes videos y archivos use las API de Imgur y Bytescale 
ya que se pueden usar sin tener que instalar nada.

Las demas funciones de una red social tambien estan como:
* Crear un usuario.
* Seguir usuarios.
* Dejar de seguir.
* Dar like a un post.
* Sacar el like.
* Abrir el perfil de un usuario (haciendo click en su nombre si estamos logueados).
* Hacer un post que puede contener una imagen, video, archivo o ser solo texto.
* Editar el post.
* Comentar los posts.
* Ver nuestros seguidores y seguidos.
* Modificar nuestro perfil.

En la carpeta db encontramos dos archivos users.json y posts.json.
La primera vez que nos logueamos hacemos un fetch a estos archivos y colocamos los usuarios y posts en el localstorage.
Todos los cambios que vengan despues, como likes, comentarios, seguidores, posts o ediciones se van a mantener en el localstorage.
Asi que si nos deslogueamos y volvemos a entrar con otro usuario todo el contenido se va a mantener.
